1. 最具有共性的问题

hexo 中我使用的NexT配置文档_config.yml

当按照教程更改完 menu 之后，神奇的是侧栏点击一直是 404 ，尝试了很多方法之后，发现 404 的界面网址后缀有 %20 的字样，原来这是空格的意思，看到很多人说这是官方 bug，需要在 menu 模块将 ||前的那一个空格删去。

2. 由于对计算机网络原理的薄弱理解引发的问题

更改域名时的域名解析

虽然现在我搞出来了，但我还是不知道原理是什么

作者是在阿里云上操作的，因为域名在那里购入

类型 填充内容 网址
CNAME www. 用户名.github.io
A 不用填 github 给出的两个 ip
A 不用填 github 给出的两个 ip

  • 更换域名网上有很多很多的教程，无疑就是解析域名使其更换位置（？重定向），在 github 上也要在setting中cutstom domain中写明自己的域名，为了方便起见，强烈建议所有的名字都是统一的。

  • 需要注意的是每次hexo提交都会覆盖 github 的 CNAME，所以需要在自己装入 hexo 的文件夹的 source 中加入一个 CNAME 文件，放入自己的域名，听过来人的经验一定不能加后缀（推荐mac OS的同学，右键点开简介，将其 txt 属性也给改掉，因为每次上传完我发现依旧是 txt）

  • 容易忽略的是创建 tags 和 categories 或者是 vim 修改时，一定要注意缩进和冒号后面的一个空格

    mac OS每次都要 sudo 就很累，可以将文件夹右击显示简介，更改为所有人可以读与写，并在下面的齿轮⚙上应用于其所包含的所有文件。
